# v0.1 Alpha Boundary

OpenSciFlow Verified Capsules v0.1 alpha is a small runnable product for checking execution evidence packages.

## What A User Can Do

After cloning this repository, a user can:

```bash
python -m pip install -e .
opensciflow-capsule list
opensciflow-capsule validate verified-capsules/mdanalysis-rmsd
opensciflow-capsule summary verified-capsules/gromacs-rmsd
opensciflow-capsule smoke verified-capsules/gromacs-rmsd
```

This is enough to inspect which capsules exist, what readiness they claim, what environment evidence has been recorded, and whether the capsule files still validate against the current schemas.

## Product Promise

v0.1 alpha promises only this:

- Capsule metadata is machine-checkable.
- Environment evidence is explicit instead of implied.
- Known blocked states can be recorded without pretending the tool works everywhere.
- Example run records can be validated against a schema.
- A local agent has a concrete artifact to inspect before deciding whether it may run a reviewed command or wrapper.

## Non-Promises

v0.1 alpha does not promise:

- A stable standard.
- Cross-HPC portability.
- CUDA, GPU, or Slurm readiness.
- Large trajectory support.
- Scientific correctness validation.
- Clinical, drug-discovery, or benchmark-performance suitability.
- Partnership with any upstream tool.

## Current Evidence

| Capsule | Current readiness | Evidence boundary |
|---|---:|---|
| `mdanalysis-rmsd` | R6 | Tiny synthetic example across local Windows and GitHub Actions Ubuntu. |
| `gromacs-rmsd` | R5 | Tiny synthetic PDB example on GitHub Actions Ubuntu. |

## Next Maturity Gates

The next version should not expand claims until it adds at least one of:

- Another independently reproduced capsule run.
- A second verified environment for `gromacs-rmsd`.
- A minimal container or Conda-lock path for one capsule.
- A reviewed Slurm wrapper tested against a real cluster policy.
- A standards crosswalk export for one run record.

# Known Failures

Known failure records are first-class evidence.

They help an agent avoid pretending that every error is new or solved by another generated shell command.

Common categories:

- environment mismatch;
- missing input;
- missing model weight;
- license or citation gap;
- HPC scheduler or module mismatch;
- file permission issue;
- unsupported container runtime;
- upstream tool bug.

Known failures should include symptoms, likely causes, and the safe next action for an agent.
